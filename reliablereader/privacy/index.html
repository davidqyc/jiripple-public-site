ReliableReader Privacy Policy

Effective Date: Pending publication

English / 简体中文

ReliableReader is an iOS app for importing long-form text and generating speech audio for listening. This policy describes the data handling behavior of the current beta version, including the optional in-app feedback system.

For privacy questions, contact: reliablereader-privacy@jiripple.com
For support questions, contact: reliablereader-support@jiripple.com

Information You Provide

ReliableReader may process text that you import, paste, type, or share into the app. This can include article titles, article body text, URLs shared from other apps, and text passed through Shortcuts.

When you use speech generation, the article text you ask the app to read is sent to the Azure Speech endpoint that you configure in the app. This is required for the text-to-speech feature.

ReliableReader does not include a shared server-side Azure key in the current beta. You provide your own Azure Speech key, region, and endpoint in the app settings.

If you actively submit feedback in the app, the feedback message you write is sent to the ReliableReader feedback backend and stored as support/feedback content. The developer may view that message and may reply through the feedback system. ReliableReader does not collect optional email, phone number, or name fields in this version.

Azure Speech Configuration

The current beta stores your Azure Speech configuration locally on your device:

  • Azure Speech key: stored in Keychain.
  • Azure region, endpoint, and credential metadata: stored in local app settings.
  • Audio quality and text-processing settings: stored in local app settings.

Azure Speech requests are sent directly from the app to the endpoint you configure. Azure's own terms and privacy practices may apply to the Azure Speech service you use.

Local Storage

The current beta may store the following information locally on your device:

  • Azure Speech configuration and related settings.
  • The current article draft or imported text, so the app can restore it after restart.
  • Generated speech audio cache, so the same content can be played again without regenerating every segment.
  • Playback speed, audio quality, text-cleaning options, diagnostics settings, and similar app preferences.
  • Diagnostic information when advanced diagnostics are enabled or copied by you.
  • Feedback session tokens in Keychain when you submit in-app feedback.

The app includes local controls for clearing saved drafts, clearing the local audio cache, and deleting the saved Azure key. Clearing local drafts or cached audio does not delete data that may already have been sent to Azure Speech for generation, and it does not delete feedback already submitted to the ReliableReader feedback backend.

Diagnostics and Feedback

ReliableReader includes optional diagnostics to help troubleshoot import, generation, playback, text cleaning, and feedback issues.

Copied diagnostics are intended to be redacted where possible, including key-like strings, tokens, authorization values, cookies, URLs, SSML, local file paths, request or response body fields, and long text snippets. This redaction is a safety measure and is not guaranteed to remove every sensitive value.

The separate cleaned reading text view may include article content. Do not share cleaned reading text unless you choose to provide that content.

In-app feedback is submitted only when you choose to send it. A feedback submission may include:

  • The feedback category and feedback message you write.
  • App version, build number, iOS version, device model, locale, and time zone.
  • Optional safe diagnostics summary fields if you choose to include them, such as feature area, playback state, safe app error code, retry count, coarse network state, import/TTS/cache state, and error count.

The in-app feedback system does not upload article body text, cleaned reading text, Azure keys, Bark tokens, authorization headers, cookies, raw request or response bodies, SSML, full URLs, complete webpage contents, clipboard contents, or local file paths. Server-side validation is intended to reject these fields if they are included accidentally.

When feedback is submitted, the backend stores feedback session metadata, the user feedback message, developer replies, status events, and optional safe diagnostics summaries. Feedback records are retained for 7 days in the current backend model, then should expire or be deleted according to the backend retention process.

The feedback system uses a sessionId and a high-entropy sessionToken so the app can check for replies without an account. The app stores the sessionToken in Keychain. The backend stores only a hash of the session token. The session token is not shown in the app UI and is not included in developer notifications.

Developer replies may be visible in the app when you refresh the feedback status. ReliableReader does not promise real-time support or that every feedback report will receive a reply.

ReliableReader may use Bark as a server-side developer notification service for new feedback. The app does not contain a Bark token. Bark notifications are limited to low-sensitivity routing and triage fields, such as a session id prefix, category, state, timestamp, app version, build number, or a developer-only URL. Bark notifications do not include your feedback message, article text, cleaned reading text, session token, Azure key, or raw diagnostics.

If you send feedback through TestFlight, email, screenshots, screen recordings, copied diagnostics, or other manual channels, the information you include will be handled through that feedback channel. Do not paste Azure keys, secrets, request headers, request bodies, private article text, cleaned reading text, or other sensitive content into feedback unless you choose to share it.

App Capabilities and Permissions

ReliableReader uses network access to reach the Azure Speech endpoint you configure. It also uses background audio playback so generated audio can continue playing according to iOS audio behavior.

The app can receive content through the iOS share sheet, Shortcuts, URL handoff, and clipboard import. Clipboard import happens after you tap the import action in the app. Shared URLs are imported as URL text in the current beta.

Based on the current app configuration, ReliableReader does not request access to the microphone, location, contacts, or photo library.

Services Not Included in the Current Beta

The current beta does not include:

  • An account system.
  • Cloud sync.
  • Advertising.
  • Third-party analytics or tracking SDKs.

The current beta does include a developer-operated backend only for feedback that you actively submit. It is not used for account login, cloud sync, advertising, analytics, tracking, hosted TTS, or storing Azure keys.

Your Choices

For beta testing, use non-sensitive text when possible. You control whether to configure Azure Speech, what text to import, when to generate speech audio, and what feedback or diagnostics to share.

You can remove local data using the controls in the app settings. Local deletion controls do not delete feedback already submitted to the feedback backend. You can also contact reliablereader-privacy@jiripple.com for privacy-related requests or questions.

Changes to This Policy

This policy may be updated as ReliableReader changes. The public version should be reviewed before each release or TestFlight expansion.
